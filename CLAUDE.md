## The game

A hidden-object game built for the Leadership Hackathon (shared stack in `STACK.md`), mascot: `Shogi-Corgi.png`
(in `public/`).

- Each round generates a scene with **15 copies of the mascot** hidden/scattered among clutter/decoys.
- Player taps/clicks all 15 within a **3-minute continuous countdown**.
- Clearing a scene does **not** end the round — it immediately generates a new scene and keeps counting
  cumulative score, continuing until the 3-minute timer runs out.
- Score = total mascots found across all scenes within the 3 minutes.
- On timeout, prompt for a username and record the score to the highscore board (Supabase `scores` table).
- **Orientation: landscape** (confirmed with the participant) — canvas/layout/controls assume the phone is
  held sideways.

## Stack (from STACK.md — shared across all hackathon participants)

| Layer | Choice |
|---|---|
| Framework | Next.js (latest, App Router) |
| Language | TypeScript |
| Database | Supabase (Postgres) — this project skipped the SQLite workshop step and went straight to Supabase per participant's request |
| Hosting | Vercel |
| Package manager | npm |

`scores` table (Postgres, via Supabase):
```sql
CREATE TABLE IF NOT EXISTS scores (
  id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  winner_name TEXT NOT NULL,
  score INTEGER NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### Constraints (from STACK.md)

- No ORM — it's one table, use the Supabase JS client directly.
- No auth/user accounts/sessions — the username field is just a public highscore label, not a login.
- Don't swap the framework/database for a "better" alternative — the point is a shared stack across
  all participants' games.

### Mobile-first requirement

Primary target is a mobile phone, not a laptop demo. Canvas/viewport/controls are sized for phone screens
in **landscape** orientation.

### Known dev gotcha

Fresh Next.js apps show a hydration warning on `<body>` in `app/layout.tsx`, almost always from a browser
extension injecting attributes pre-hydration. Fix by scoping `suppressHydrationWarning` to `<body>` only
(not the whole tree, so real hydration bugs elsewhere still surface):
```tsx
<body className="..." suppressHydrationWarning>{children}</body>
```
